## Sandbox 3.0

We're happy to announce the latest Sandbox update!

![](/img/blog/2026-january/sandbox.webp)

<MediaLegend>Sandbox Cover</MediaLegend>

This update brings lots of improvements to the Sandbox game mode: new features, entities, tools, and a big polish pass on the UI and existing systems. It's heavily focused on improving the user experience and making it more fun (and easier) to create content, entities, and tools.

### Context Menu

The Context Menu was the biggest change in this update!


#### Selecting Entities

You can now **Select** and **Modify** Entities by clicking on them while the Context Menu is open.

<VideoExternal path="/blog/2026-january/sandbox-selecting-balloons.webm" />

<MediaLegend>Sandbox game-mode Context Menu selecting entities</MediaLegend>

Each entity has its own set of properties that can be modified through the Context Menu. You can configure it programmatically in your custom class by defining the [`selected_context_menu_items`](/docs/next/explore/sandbox-game-mode/context-menu#when-selecting-an-entity) property.

Now a new "preview mesh" will appear when using *spawner* Tool Guns, showing a **ghost preview** of the entity you're about to spawn:

<VideoExternal path="/blog/2026-january/sandbox-thrusters.webm" />

<MediaLegend>Sandbox game-mode customizing Thrusters</MediaLegend>


<details>
  <summary>Adding a property to Thruster Entity</summary>

For example, the Thruster custom particle can be set like this to display on Context Menu:

```lua title="Client/Thruster.lua"
Thruster.selected_context_menu_items = {
	{
		id = "thruster_particle_asset",
		type = "select",
		label = "particle",
		options = Thruster.assets_list,
		callback = function(value)
			ContextMenu.selected_entity:CallRemoteEvent("SetParticleAsset", value)
		end,
		value = function()
			return ContextMenu.selected_entity:GetValue("Particle")
		end,
	},
	...
}
```
</details>

<details>
  <summary>Adding Description and Tutorials to a Tool</summary>

You can add custom description and tutorials easily like this, it is automatically picked up by the Tutorials system:

```lua title="Client/ColorGun.lua"
-- Tool Description
ColorGun.description = "Paint props and objects with a selected color"
```

### New Entities

We added a bunch of fun new entities to Sandbox, and there's more on the way!

#### Retro Camera

The `RetroCamera` is a *polaroid* instant camera that lets you take photos in real time.

<VideoExternal path="/blog/2026-january/sandbox-retro-camera.webm" />

<MediaLegend>Sandbox game-mode new Retro Camera Entity</MediaLegend>

It uses a <Classes.SceneCapture /> and outputs a base64 image which is printed immediately onto a photo film, that is synchronized and persistent in the server!


#### Button & Wire

The `Button` is a new entity that's *pressable*, **activating** and **deactivating** other entities:

<VideoExternal path="/blog/2026-january/sandbox-button-lamp.webm" />

<MediaLegend>Sandbox game-mode new Button Entity and Wire Tool Gun</MediaLegend>

You can connect the button to other entities using the new `Wire Tool Gun`.

<VideoExternal path="/blog/2026-january/sandbox-button-gadgets.webm" />

<MediaLegend>Sandbox game-mode new Button Entity activating several devices</MediaLegend>

It works by keeping a list of "connected" entities and calling `:Activate()` and `:Deactivate()` on them, if they implement those methods.

Since some Sandbox entities already implement those methods, you can wire the button to them and create cool integrations.

You can create your own entities and implement these two methods to make it work:

```lua title="Server/MyEntity.lua"
function MyEntity:Activate()
	-- Turns on!
end

function MyEntity:Deactivate()
	-- Turns off!
end
```

### Refactored APIs

Now, all Sandbox systems are consolidated and exported into a single global variable: `Sandbox`. You can access all Sandbox systems and methods through it from your scripts, for example [`Sandbox.ContextMenu`](/docs/next/explore/sandbox-game-mode/context-menu) or [`Sandbox.SpawnMenu`](/docs/next/explore/sandbox-game-mode/spawn-menu).

We also created and exposed several new subsystems such as [`Tutorials`](/docs/next/explore/sandbox-game-mode/extra-features#tutorials), [`Notifications`](/docs/next/explore/sandbox-game-mode/extra-features#notifications), [`SpawnHistory`](/docs/next/explore/sandbox-game-mode/extra-features#spawnhsitory), and even exposed the WebUI as `Sandbox.WebUI`.

## Interaction ToolTips

We added a brand new **Interaction ToolTip** system for <Classes.Prop />, <Classes.Weapon /> (<Classes.Pickable />), and <Classes.Vehicle />. A tooltip (text + image) appears on screen when you look at those interactable objects:

<VideoExternal path="/blog/2026-january/tooltips.webm" />

<MediaLegend>Tooltips on screen when hovering interactable objects</MediaLegend>

For **Props**, you can now change the tooltip text using <MethodReference type="Class" class_name="Prop" method="SetInteractionToolTipText" show_class_name />. This lets you set custom messages like "Open", "Close", or "Press" for interact-only Props.

You can disable this system with the new static method <MethodReference type="StaticClass" class_name="Viewport" method="SetInteractionToolTipEnabled" show_class_name is_static />.


## Scripting & API


### New Class: TextRender

We already had a **TextRender** class, but it was actually a **Text3D** component (from the Unreal Text3D Plugin).

We then decided to create a new class <Classes.TextRender /> and rename the old TextRender to <Classes.Text3D />. This gives us a new lightweight class for rendering simple 2D text with a font, while keeping **Text3D** separate with its own methods (as before).

This new **TextRender** class also supports custom fonts via <MethodReference type="Class" class_name="TextRender" method="SetFont" show_class_name />.

*For this upgrade, we introduced a new [Compatibility Version 1.103](/docs/next/core-concepts/packages/compatibility-versions#version-1103) so we don't break any existing scripts using the old TextRender class.*


### Texture Parameters

Texture parameters got a big flexibility boost: any texture parameter now supports loading **Base64**, **URL images** (experimental), and **cooked** texture assets.

![](/img/blog/2026-january/custom-images.webp)

<MediaLegend>Custom Web Images loaded as textures in runtime</MediaLegend>

You can directly set the URL/Base64 to materials using the <MethodReference type="Class" class_name="Paintable" method="SetMaterialTextureParameter" show_class_name is_base />!


### Cables & Constraints

<Classes.Cable /> was heavily improved. "Locked" constraints will **no longer** be created with an automatically calculated offset. A new parameter was added to `Cable.AttachEndTo()`, `constraint_offset`, so you can customize the offset through scripting and have more control over constraints.

We also added a bunch of new methods to customize physics constraint and motor settings, such as <MethodReference type="Class" class_name="Cable" method="SetLinearMotorPositionSettings" /> and <MethodReference type="Class" class_name="Cable" method="SetAngularMotorSLERPOrientationSettings" />. See them all in [the next blog section](#cables-motor--physics-constraint).

The Cables spawn performance was also improved: if you use an **All-Free** constraint, the Physics Constraint now stays deactivated (great when you only want visuals). We also made small improvements to stabilize physics jitter.

## Quality of Life Improvements


### WebUI Inputs

Besides the new <MethodReference type="Class" class_name="WebUI" method="HasNodeFocus" show_class_name /> method, now pressing Escape while a WebUI node has focus, causes the node to unfocus (instead of opening the escape menu).

Also, while a WebUI node has focus, **all game input is ignored**. This makes in-game forms and text inputs feel much more "normal", and avoids those annoying cases where the player types into a UI and also triggers gameplay binds.


### File Transfer Speeds

We iterated again on the download system, and file transfer speeds are now improved for high-ping players, making joining servers feel smoother in worse network conditions.


### VOIP Network Usage

On VOIP, data is now only transmitted to players within ~30 meters (when using local channel), reducing unnecessary bandwidth usage.


### Drop Input

Pressing the Drop input (`G`) now drops the grabbed prop as well.

## Conclusion

The Sandbox game-mode is finally taking the shape I wanted for it, becoming really next to my inspiration Garry's Mod version of it. I will have some work to do and expand, but we are much closer than before.

My goal with the Sandbox is for it to become a fun game-mode, that can be an easy source of new integrations and entities. But also a source of good examples of how to implement custom entities, tools and systems for nanos world.

It's always great when I dedicate time to writing scripts and game-modes, besides it being hell of fun, I always find missing methods, problems and I end up integrating and fixing stuff during this period.

I'm all the time focused on improving what we already have and fixing chronic problems, and this month it was possible to add many cool features to the game and the scripting API!
